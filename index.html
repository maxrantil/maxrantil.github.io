Projects

Shell Creation Project: In this group project, we developed a POSIX-compliant shell from scratch. This was a challenging task that required a deep understanding of Unix and POSIX functionalities, including command execution, environment management, and interaction with shell variables. We used C programming language for this project, and it helped me to improve my skills in system and network administration, project management, and building complex systems. The final product was a robust shell interpreter, 42sh, that supports a broad spectrum of UNIX and POSIX functionalities.

Roger-Skyline-1: This project involved setting up a Linux-based server on a virtual machine, configuring system settings, and installing the Artix operating system. Key tasks included establishing a secure shell (SSH), configuring a firewall, implementing Fail2Ban, and setting up an Apache web server. I also developed scripts for regular package updates and crontab monitoring, configured a local mail server, and set up a static IP. This project showcased my skills in server setup, network administration, and security measures.

ft_printf: The ft_printf project involved recreating the printf function in C using a limited set of built-in C Library functions. The recreated function, ft_printf, supports various conversion specifiers, flags, field widths, precisions, and length modifiers. This project served as a valuable exercise in understanding variadic functions in C and provided a practical example of a basic "dispatcher" in C. It showcased my skills in working with low-level functions, memory management, and complex function creation in C.

Game of Life: The Game of Life project involved implementing Conway's Game of Life, a cellular automaton, with an emphasis on optimizing the performance. The game operates on a two-dimensional grid of cells, each transitioning between states based on its neighbors. In this group project we incorporated significant optimizations, such as halting iterations upon detecting repeating patterns, using bitwise operations for efficient state checks and predictions. Additionally, we developed an interactive graphical user interface for the game, enhancing user engagement. This project highlighted my skills in algorithm optimization, bitwise operations, and GUI development.

ft_ls: The ft_ls project involved recreating the Unix 'ls' command, providing a deep dive into Unix file systems, file permissions, and metadata. My version, ft_ls, supports options like long format listing, recursive listing, hidden files inclusion, file order reversal, and sorting by modification time. This project highlighted my proficiency in C programming, understanding of Unix systems, and ability to handle complex data structures and system functions.

libft: The Libft project was a foundational project in the 42 curriculum, requiring us to recreate several standard C library functions from scratch. The project served as an introduction to C programming, emphasizing the understanding of data types, control structures, and basic algorithms. The recreated library, libft, includes a variety of functions, such as those for string manipulation, memory allocation, and character checks, as well as more complex data structures like linked lists. This project was instrumental in honing my skills in C programming, memory management, and algorithm implementation, setting the stage for more advanced projects in the curriculum.
